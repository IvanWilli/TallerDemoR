Unidad 4 Lectura

En la unidad previa caracterizamos un hogar censal. En esta unidad trabajaremos la base de datos censal, con todas sus viviendas, hogares y personas. Para esto recorreremos distintos paquetes del entorno tidyverse, cuyo fin es generar un entorno de herramientas especializada en el análisis de datos.

# install.packages("tidyverse")
library("tidyverse")
library(fontawesome)
library(knitr)

Pero primero haremos un repaso por las distintas opciones de lectura de datos que nos ofrece R. Creemos una carpeta que se llame “unidad2” y una subcarpeta que se llame “Data”, donde dejemos las bases de datos disponibles como materiales de la unidad. Verifiquemos que tengamos todas las bases de datos correspondientes en la carpeta “Data” mediante list.files().

list.files("unidad2/Data")

Para mostrar las diversas formas de lectura de datos que ofrece R nos valdremos de la Encuesta Permanente de Hogares (EPH), un operativo estadístico llevado adelante por INDEC con motivo de caracterizar la evolución de los mercados de trabajo de 31 aglomerados urbanos del país. Por ejemplo, si quisiéramos leer la base de datos del primer trimestre de 2019, ¿qué opciones tenemos? Es muy importante aquí que la ruta del archivo a leer sea a partir de la posición de trabajo actual (recuerda getwd) y hacia donde apuntar (utiliza el predictor con tab al tipear las comillas).

  • Texto plano con separador ;
# R base trae funciones para la lectura de datos, las más comunes son "read.table" y "read.csv"
EPH_1t19_txt <- read.table(file = "Data/usu_individual_t119.txt", header = T, sep = ";")
  • Un excel con extensión xls o xlsx, seleccionando la pestaña (y el rango de celdas si queremos). Aprovechemos a leer la documentación de esta función: ?read_xls o desde el tab de paquetes en RStudio.
library(readxl) # pertenece a tidyverse
EPH_1t19_xls <- read_xls(path = "Data/usu_individual_t119.xls", sheet = 1)

# quiero el rango B3:C5
read_xls(path = "Data/usu_individual_t119.xls", sheet = 1, range = "B3:C5", col_names = FALSE)
  • También podemos leer extensiones .dbf, SAS, SPSS, Stata, etcéteras… gracias a la librería foreign. Por ejemplo, una base de datos del entorno SPSS:
library(foreign)
EPH_1t19_spss <- read.spss("Data/usu_individual_t119.sav", to.data.frame=TRUE)
  • En la primer unidad dijimos que un paquete es compartir, y es lo que hace a una comunidad dinámica. Ya existe un paquete llamado eph. Permite no solo leer las bases de datos trimestre a trimestre, sino funciones para obtener muchos resultados y panelear olas.
library(eph)
EPH_1t19_eph = get_microdata(year = 2019, trimester = 1, type = 'individual')
  • También podemos leer un archivo online, por ejemplo la base de microdatos de DEIS (Dirección de Estadísticas e Información en Salud). Vayamos a la web primero y veamos como obtener el link. Tiene extensión csv. Utilicemos la función read.csv comentada anteriormente, pero en vez de apuntar al archivo local hagámoslo al url:
url <- "http://www.deis.msal.gov.ar/wp-content/uploads/2019/01/DefWeb17.csv"
DEf17 <- read.csv(url, header = T)

COVID19 data: Si querés estudiar la evolución de los casos nuevos y las defunciones registradas por covid19 podés descargar online la base, que esta disponible de manera libre por MSAL (aplausos). Está contenida en un archivo “zip”. La descargamos y la descomprimimos con la ayuda del paquete zip,luego la leemos como cualquier csv (en este caso las primeras 1000 filas):
library(zip)
url <- 'https://sisa.msal.gov.ar/datos/descargas/covid-19/files/Covid19Casos.zip'
download.file(url, "Covid19Casos.zip")
unzip("Covid19Casos.zip")
covid_data <- read.csv("Covid19Casos.csv", fileEncoding = "UTF-8", nrows = 1000)

Pero seguro ya estas cansado de levantar datos: “Dame el censo” se escucha murmurar en el aula-zoom.

Trabajaremos con una muestra aleatoria de datos que provienen de un proyecto de investigación expuesto en la conferencia AEPA17, y que tiene como objetivo traducir a microdato cualquier base con formato REDATAM, y cuyos resultados para el censo 2010 de Argentina pueden verse aquí. La base allí disponible NO es oficial, por lo que sus resultados pueden no ser los publicados, pero con motivo del taller nos permite jugar con las herramientas de manipulación de datos a nivel de máxima desagregación.
